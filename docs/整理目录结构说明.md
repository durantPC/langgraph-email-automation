# 项目目录结构整理方案

## 整理目标

将散落在根目录的文件按功能分类整理到相应目录，使项目结构更清晰。

## 整理计划

### 1. 用户数据文件 → `data/users/`
- `user_data.json` → `data/users/user_data.json`
- `user_email_data_*.json` → `data/users/user_email_data_*.json`
- `username_mapping.json` → `data/users/username_mapping.json`

### 2. 数据库脚本 → `scripts/database/`
- `create_index.py` → `scripts/database/create_index.py`
- `create_index_1024.py` → `scripts/database/create_index_1024.py`
- `create_index_2560.py` → `scripts/database/create_index_2560.py`
- `create_index_4096.py` → `scripts/database/create_index_4096.py`
- `clear_db.py` → `scripts/database/clear_db.py`

### 3. 启动脚本统一
- 保留 `scripts/` 目录下的启动脚本
- 根目录的 `start.bat` 可以保留作为快捷方式，或移动到 `scripts/`

### 4. 需要修改的代码文件
- `backend_api.py` - 更新用户数据文件路径
- `create_index*.py` - 更新数据路径引用（如果有）

## 整理后的目录结构

```
langgraph-email-automation/
├── backend_api.py
├── main.py
├── main_continuous.py
├── requirements.txt
├── README.md
│
├── src/                    # 后端核心代码
├── frontend/               # 前端项目
├── docs/                   # 项目文档
│
├── data/                   # 数据目录
│   ├── agency.txt         # 知识库文档
│   └── users/             # 用户数据（新建）
│       ├── user_data.json
│       ├── user_email_data_*.json
│       └── username_mapping.json
│
├── db/                     # 向量数据库
├── db_1024/
├── db_2560/
└── db_4096/
│
└── scripts/                # 脚本目录
    ├── start_all.bat
    ├── start_backend.bat
    ├── start_frontend.bat
    └── database/           # 数据库脚本（新建）
        ├── create_index.py
        ├── create_index_1024.py
        ├── create_index_2560.py
        ├── create_index_4096.py
        └── clear_db.py
```
